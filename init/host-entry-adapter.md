# Host Entry Adapter

## 目标

定义 `CommonHE/init` 在不同宿主环境下，如何判断和生成正确的 agent 启动入口。

## 背景

不同工具或工作流环境，可能使用不同的入口机制。

例如：

- 某些环境读取项目根目录 `AGENTS.md`
- 某些环境读取特定协作目录
- 某些环境允许多个入口并存

因此初始化器不能把某一种入口机制写死。

## 建议适配流程

### 第一步：识别宿主环境

初始化器应先判断：

1. 当前工具生态是否明确要求某个入口文件
2. 当前项目是否已经存在入口文件
3. 是否存在现成协作目录，如 `.codex/`、`.claude/` 或其他约定目录

### 第二步：确定主入口

根据宿主环境选择主入口。

例如：

- 若当前环境要求根目录 `AGENTS.md`
  - 则生成并维护 `<project-root>/AGENTS.md`

- 若当前环境要求其他入口
  - 则生成相应入口，并把其他文件作为辅助结构

### 第三步：生成辅助结构

即使主入口不同，也可以继续生成：

- `.codex/agents/`
- `docs/`
- `workflow/`

但生成前提是这些结构不会与宿主环境冲突。

## 对初始化器的要求

初始化器至少要输出这三类结论：

1. 当前识别到的宿主入口机制
2. 准备生成的主入口文件
3. 其他辅助目录是否同时生成

## 针对当前已知场景的说明

以当前 Codex 工作方式为例，项目根目录 `AGENTS.md` 很可能就是关键入口之一。

因此当 `CommonHE` 复制到类似项目时，初始化器应优先支持根目录 `AGENTS.md` 生成策略。
