# {{project_name}} 评分标准

## 通用维度

以下维度适用于所有任务：

| 维度 | 默认阈值 | 权重 | 说明 |
|-----|---------|------|------|
| 功能完整性 | >= 7 | 高 | 是否满足 Contract 功能需求 |
| 代码质量 | >= 6 | 中 | 可读性、可维护性、架构合理性 |
| 规格合规 | >= 7 | 高 | 是否忠于 Contract 和上游规格 |
| 测试覆盖 | >= 6 | 中 | 测试质量和覆盖度 |

## 前端扩展维度

以下维度在涉及 UI/UX 时启用：

| 维度 | 默认阈值 | 权重 |
|-----|---------|------|
| 设计质量 | >= 6 | 低 |
| 原创性与工艺 | >= 5 | 低 |
| 响应式与可访问性 | >= 6 | 低 |

## 后端扩展维度

以下维度在涉及 API/服务端时启用：

| 维度 | 默认阈值 | 权重 |
|-----|---------|------|
| API 契约合规 | >= 7 | 中 |
| 性能 | >= 6 | 低 |
| 安全性 | >= 7 | 高 |
| 错误处理 | >= 6 | 中 |
